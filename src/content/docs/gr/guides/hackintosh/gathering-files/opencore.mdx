OpenCore είναι ουσιαστικά ένα πρόγραμμα που εκτελείται από το firmware (το BIOS/UEFI) και ξεκινά το macOS.

Έχει τη δυνατότητα να εφαρμόζει διορθώσεις, να προσθέτει οδηγούς και/ή να παραποιεί τις πληροφορίες της συσκευής εκ των προτέρων.

## Κατέβασμα του OpenCore

Ας ξεκινήσουμε επισκεπτόμενοι το αποθετήριο του στο GitHub, που βρίσκεται [εδώ](https://github.com/Acidanthera/OpenCorePkg).

<Aside type="tip">
Ένα αποθετήριο Git είναι ουσιαστικά ένα αρχείο κώδικα, που περιέχει διαφορετικές εκδόσεις του. Το Git δεν είναι απαραίτητα το GitHub. Το GitHub είναι απλώς μια εκτεταμένη και γραφική διεπαφή του. Αν θέλετε να μάθετε περισσότερα για το Git, αναζητήστε το στο διαδίκτυο καθώς είναι ένα πολύ ευρέως χρησιμοποιούμενο εργαλείο στην ανάπτυξη λογισμικού.
</Aside>

![Αρχική σελίδα του αποθετηρίου OpenCorePkg](~/assets/OpenCorePkg.png)

Μεταβείτε στην ενότητα των εκδόσεων. Προφανώς, εδώ βρίσκονται οι εκδόσεις του έργου. Εδώ μπορούμε να βρούμε τα διαφορετικά αρχεία κάθε έκδοσης και ίσως το αρχείο αλλαγών. Αυτή τη στιγμή μας ενδιαφέρουν μόνο τα αρχεία κατασκευής που σχετίζονται με την τελευταία έκδοση.

<Aside type="tip">
Ένα αρχείο κατασκευής είναι το τελικό αποτέλεσμα του κώδικα μετά τη μεταγλώττιση, μια διαδικασία που περιλαμβάνει τη “μετατροπή” του κώδικα που διαβάζεται από τον άνθρωπο σε μια γλώσσα που κατανοούν οι υπολογιστές, που ονομάζεται μηχανικός κώδικας.
</Aside>

![Αρχική σελίδα του αποθετηρίου OpenCorePkg με επισημασμένη την ενότητα των εκδόσεων](~/assets/OpenCorePkgReleaseSection.png)

Η τελευταία έκδοση βρίσκεται πάντα στην κορυφή. Στο κάτω μέρος της “κάρτας” της έκδοσης βρίσκονται τα αρχεία που σχετίζονται με αυτήν την έκδοση. Μπορείτε να δείτε τόσο τις εκδόσεις RELEASE όσο και DEBUG.

<Aside type="tip">
Μια βελτιστοποιημένη κατασκευή, γνωστή και ως έκδοση release, είναι ένας τύπος κατασκευής όπου ο κώδικας βελτιστοποιείται στη διαδικασία μεταγλώττισης, επιταχύνοντας την εκτέλεση και αφαιρώντας τις πληροφορίες αποσφαλμάτωσης. Αντίθετα, μια μη βελτιστοποιημένη κατασκευή debug περιέχει περισσότερες πληροφορίες και καταγραφές αποσφαλμάτωσης, αλλά είναι σημαντικά πιο αργή από μια βελτιστοποιημένη κατασκευή. Χρησιμοποιήστε κατασκευές debug μόνο όταν αντιμετωπίζετε προβλήματα.
</Aside>

![Σελίδα εκδόσεων του OpenCorePkg](~/assets/OpenCorePkgReleasePage.png)

Θα κατεβάσουμε την κατασκευή debug προς το παρόν, ώστε να μπορούμε να διαγνώσουμε τυχόν προβλήματα που μπορεί να προκύψουν αργότερα στη διαδικασία. Κάντε κλικ (στην κατασκευή debug) για να ξεκινήσει η λήψη.

## Οργάνωση των αρχείων και προετοιμασία του φακέλου EFI

Μόλις ολοκληρωθεί, εξαγάγετε τα περιεχόμενα κάπου οργανωμένα ώστε να μπορείτε να παρακολουθείτε τα πάντα.

Τα περιεχόμενα του φακέλου Drive θα αντιγραφούν αργότερα στο USB σας. Αντιγράψτε αυτόν τον φάκελο EFI από το OpenCorePkg/X64/EFI. Είναι το ίδιο το OpenCore.

Στη συνέχεια, αντιγράψτε το Sample.plist στο EFI/OC με το όνομα Config.plist. Αυτή θα είναι η διαμόρφωση του OpenCore που θα επεξεργαστούμε για το σύστημά μας αργότερα.

Θα πρέπει τώρα να έχετε μια διάταξη παρόμοια με την ακόλουθη.

<FileTree>
- OCStuff
  - OpenCorePkg
    - …
  - Δίσκος
    - EFI
      - OC
        - OpenCore.efi
        - **Config.plist**
        - …
      - …
    - …
  - …
</FileTree>

Με αυτά κατά νου, συγχαρητήρια! Τώρα ξέρετε τι είναι το OpenCore, πώς να περιηγηθείτε στη διεπαφή χρήστη του GitHub για να κατεβάσετε την κατασκευή ενός έργου από τη σελίδα εκδόσεων και πώς να διατηρήσετε την περιπέτειά σας με το hackintosh οργανωμένη.
