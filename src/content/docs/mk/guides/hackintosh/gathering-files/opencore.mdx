OpenCore е програма што го извршува фирмверот (BIOS/UEFI) кој го стартува macOS.

Има можност да примени петчовите, да додава драјвери и/или да ги измами информациите за уредот претходно.

## Симнување OpenCore

Ајде да почнеме со посета на OpenCore GitHub-от, лоцирано [тука](https://github.com/Acidanthera/OpenCorePkg).

<Aside type="tip">
Git во основа е архива на код, која содржи различни ревизии на него. Git не е нужно GitHub. GitHub е само продолжен и графички интерфејс за него. Ако сакате да дознаете повеќе за Git, пребарајте го на интернет бидејќи е многу широко користена алатка во софтверски девелопмент.
</Aside>

![OpenCorePkg repository homepage](~/assets/OpenCorePkg.png)

Одете во "Release" Делот. Тоа се изданијата на проектот. Овде можеме да ги најдеме различните датотеки на секое издание и можеби да го смениме дневникот. Ние сме заинтересирани само за градежните артефакти поврзани со најновиот "Release" во моментов

<Aside type="tip">
Изградениот артефакт е крајниот резултат на кодот по компилацијата, процес кој вклучува „конверзија“ на код читлив од луѓето во јазик разбирлив од компјутерите, наречен машински код.
</Aside>

![OpenCorePkg repository homepage with release section highlighted](~/assets/OpenCorePkgReleaseSection.png)

Најновото издание секогаш се наоѓа на врвот. На дното на „картичката“ на изданието се наоѓаат датотеките поврзани со ова издание. Можете да ги видите двете изданија "RELEASE" и "DEBUG".

<Aside type="tip">
Оптимизирана градба, е тип на градба каде кодот е оптимизиран во процесот на компилација, забрзувајќи го извршувањето и отстранувајќи ги информациите за дебагирање. Спротивно на тоа, неоптимизираната верзија за отстранување грешки содржи повеќе информации за отстранување грешки, но е значително побавна од оптимизирана верзија. Користете "DEBUG" за отстранување грешки само кога решавате проблеми.
</Aside>

![OpenCorePkg release page](~/assets/OpenCorePkgReleasePage.png)

Засега ќе го искористиме build-от за отстранување грешки, така што ќе можеме да ги дијагностицираме сите проблеми што може да се појават подоцна во процесот. Кликнете (на "DEBUG") за да започне преземањето.

## Организирање на фајловите и подготовка на папката EFI

Откако ќе заврши, извадете ја неговата содржина некаде организирано за да можете да следите сè.

Содржината на папката „Drive“ подоцна ќе се копира на вашиот USB. Копирајте го `EFI` фолдерот од `OpenCorePkg/X64/EFI`. Тоа е самиот OpenCore.

После, копирајте `Sample.plist` во `EFI/OC` со `Config.plist` како името. Ова ќе биде вашата конфигурација OpenCore која ќе биде уредена за нашиот систем понатаму.

Сега треба вака да ви изгледа од прилика:

<FileTree>
- OCStuff
  - OpenCorePkg
    - …
  - Drive
    - EFI
      - OC
        - OpenCore.efi
        - **Config.plist**
        - …
      - …
    - …
  - …
</FileTree>

Браво! Сега знаете што е OpenCore, како да се движите низ интерфејсот на GitHub за да ја преземете верзијата на проектот од страницата за објавување и како да ја организирате вашата хакинтош авантура.
